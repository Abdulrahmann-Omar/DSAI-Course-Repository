# Contributing Guidelines

Thank you for your interest in contributing to this DSAI Course Repository!

## How to Contribute

### Reporting Issues
- Use the GitHub Issues tab to report bugs or suggest improvements
- Provide clear descriptions and steps to reproduce any issues

### Adding Course Materials
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/add-course-materials`)
3. Follow the established folder structure
4. Ensure all notebooks are cleared of output or have clean outputs
5. Add appropriate documentation
6. Submit a pull request

### Code Standards
- Follow PEP 8 for Python code
- Include docstrings for functions and classes
- Add comments for complex logic
- Use descriptive variable names

### Notebook Standards
- Clear markdown headers for each section
- Include imports at the top
- Add explanatory markdown cells
- Ensure reproducibility (set random seeds)

## Review Process
1. All contributions are reviewed by maintainers
2. Feedback will be provided within 7 days
3. Once approved, changes will be merged

## Questions?
Open an issue or reach out to the repository owner.
